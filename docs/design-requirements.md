| 副 | 新成人・保護者 | 着付け、成人式セット |

1. 認知（「sins酸性ストレートとは？」「ヘッドスパは？」など検索流入）
2. 比較（パーマ3種の違いなど）
3. 検討（料金・所要時間の確認）
4. 決定（電話 / LINE で予約）

## 4. ビジュアルデザイン

### 4.1 カラー

#### ブランドカラー
| 役割 | カラー | 用途 |
| --- | --- | --- |
| プライマリ | 青（例: #1d4ed8） | ボタン、見出し下線、リンク、sinsページ |
| セカンダリ | 緑（例: #22c55e） | LINEボタン |
| アクセント | ティール（例: #0d9488） | パーマページ1カード目 |
| ハイライト | パープル（例: #9333ea） | パーマページ2カード目、着付けページ |

#### ページ別アクセント（ページヘッダーのグラデーション）
| ページ | アクセント色 |
| --- | --- |
| sins / パーマ / スタッフ | 青 |
| ドライヘッドスパ | エメラルド |
| まつ毛パーマ | ピンク |
| 着付け | パープル |

#### セマンティックカラー
- 成功 / LINE: 緑
- 警告: 淡いイエロー背景 + ダークイエロー文字（注意事項ボックス）
- エラー / 免責: 赤字（ブリーチ毛NG、追加料金など）
- 情報: 青（FAQのQマーク）

#### 中立色
- 背景: 白 / 薄グレー（交互に使って区切る）
- 本文: ミディアムグレー
- 注釈: ライトグレー
- 見出し: ダークグレー（**原則すべての見出し色を統一する**。ページごとに色変えしない）

#### グラデーション
- ボタン: プライマリ色の濃淡で上→下の縦グラデ
- セクション背景: 2色の同系色で左→右（例: 青→紫）
- ページヘッダー: 淡色→白で上→下

### 4.2 タイポグラフィ

#### フォント
- 日本語: システムデフォルト（機種依存）
- 英数字: システムデフォルト
- ※リデザイン時にWebフォント指定の余地あり（推奨: Noto Sans JP / Zen Kaku Gothic New など）

#### サイズ・ウェイト
| レベル | 用途 | サイズ | ウェイト |

| H1 | ページタイトル | 36〜48px（モバイル36 / PC48） | Bold |
| H2 | セクションタイトル | 30px | Bold |
| H3 | カード・グループタイトル | 24px | Bold |
| H4 | サブセクション | 20px | Semibold |
| 本文 | 段落 | 16px | Regular |
| リード | サブタイトル、導入文 | 18px | Regular |
| 注釈 | 補足 | 14px | Regular |
| 微小注釈 | 免責 | 12px | Regular |

#### セクションヘッダー装飾
- タイトル下に **短い横線（幅96px・太さ4px・プライマリ色）を中央配置** する
- サブタイトルがある場合はタイトル下に配置し、横線はさらにその下

#### 行高
- 長文説明: ゆったりめ（1.7〜1.9）
- 見出し: やや詰め気味（1.2〜1.3）

### 4.3 レイアウト

#### コンテナ
- 最大幅: 1200px程度を想定、中央寄せ
- 左右余白: モバイル16px / デスクトップ自動

#### セクションの縦余白
- 本文セクション: 上下64px（全ページで統一）
- 重要セクション: 上下80px

#### グリッド
- 2列: 施術例、2カード比較
- 3列: 魅力紹介、ギャラリー
- グリッド間隔: 16px / 24px / 32px を用途により使い分け

#### ブレークポイント
| 名称 | 幅 | 主な変化 |
| --- | --- | --- |
| モバイル | 〜767px | 1〜2列、ハンバーガーメニュー |
| タブレット | 768〜1023px | グリッド2〜3列、料金表が横並びに |
| デスクトップ | 1024px以上 | ナビゲーション展開、グリッド3列以上可 |

### 4.4 UI部品

#### ボタン
- 形状: ピル型（完全な角丸）
- サイズ: S（小）/ M（標準、縦12px・横24px程度）/ L（大）
- 種類:
  - **プライマリ（予約/詳細）**: 青のグラデ、白文字、強い影
  - **セカンダリ（サブ）**: 白地、青枠線、青文字
  - **LINE**: 緑のグラデ、白文字
  - **電話**: プライマリと同じ見た目、角丸は緩め（矩形に近いピル型）
- ホバー: **上に約2px浮上 + 影を強める**（所要200ms程度）

#### カード
- 形状: 大きめの角丸（16〜24px）+ 柔らかい影
- パディング: 16 / 24 / 32pxを使い分け
- 特集カード: より大きな角丸（24〜32px）+ 強めの影

#### サービス詳細カード（パーマ・着付けで使用）
- 上部にカラーヘッダー（サービスのアクセント色、白文字）
- 「おすすめ」バッジは半透明白で配置
- 下部にコンテンツエリア（白背景）

#### 情報ボックス
- 角丸、色付き薄背景（例: 薄い青 / 薄いピンク / 薄いエメラルド）
- 「所要時間」「こんな方に」など補足情報の表示用

#### リストアイテム
- **チェックリスト**: 先頭アイコン付き
  - 標準は ✓（緑）
  - 装飾的用途では ♢ / ◆ / • を使用
  - アイコン色はコンテキストに合わせて変える（青 / ティール / グレー）
- **料金行**:
  - カード内: 商品名 … 価格（ドット線で区切り、価格はアクセント色）
  - 詳細ページ: 商品名と価格を左右に配置し、下線または薄灰地で区切る

#### 施術フロー（ステップ）
- 番号入り円バッジ（アクセント色、白文字）+ タイトル + 説明
- 縦に並べる

#### ギャラリーカード
- 画像エリア（薄灰プレースホルダー）+ タイトル + 説明
- アスペクト比: 16:9 / 3:4 / 1:1 を使い分け

#### FAQ
- Q: 青の円形バッジ + 白文字
- A: 緑の円形バッジ + 白文字
- カードの中にQブロックとAブロックを積む

#### 注意事項ボックス
- 淡いイエロー背景 + 濃いイエロー枠線
- タイトルは太字、中にリスト

### 4.5 アイコン
- インラインの記号装飾（✓ / ♢ / ◆ / •）を多用
- SVGアイコン使用箇所: ハンバーガーメニュー、FAQのQ/Aバッジ、魅力紹介（目・時計・スマイル）、スタッフのプレースホルダー
- アイコンセットは統一されておらず、**リデザイン時に統一候補**（推奨: Heroicons / Phosphor Icons / Lucide など）

### 4.6 画像

#### 形式と配信
- すべてAVIF形式
- 外部CDN（`images.success-salon.haton14.com`）経由で配信

#### 配置パターン
- 単体大判: 角丸大（16〜24px）+ 影、16:9比率
- 2列グリッド: 施術例、外観内観
- 3列グリッド: ギャラリー、お客様の声
- 画像の角丸はグリッド内では小さめ（12px）、単体では大きめ（16〜24px）

#### 必要な画像カテゴリ（リデザインでも継続必要）
- 店舗: 外観、内観
- sins施術例: Before/After、ショート、ミディアム×5、ロング×4
- ヘッドスパ: 施術中×2、個室、お客様の声×3、資格証×2、可動域施術×5
- ※スタッフページは現状プレースホルダー。**顔写真 or イメージ写真の差し込み候補**
- ※パーマ・まつ毛・着付けページのギャラリーも現状プレースホルダー相当。**実写差し込み候補**

## 5. インタラクション

### 5.1 スクロール挙動
- **スムーススクロール**: 全アンカーリンクで実装。ヘッダー固定分のオフセットを考慮
- **ヘッダーの変化**: スクロール100pxを超えたら影を強めて浮かせる
- **セクションのフェードイン**: ビューポートに入ったら下からふわっと表示（約600ms、opacity 0.3→1、20px↑移動）

### 5.2 ホバー
- ボタン: 浮上 + 影強調（200ms）
- ナビリンク: 色変化（グレー→プライマリ青）

### 5.3 モバイルメニュー
- ハンバーガータップで展開、ナビ項目タップで自動的に閉じる

## 6. レスポンシブ

### 6.1 モバイル優先
- 1列レイアウトが基本
- タップターゲットは十分な大きさ（44px以上推奨）
- ハンバーガーメニュー標準装備

### 6.2 タブレット以降
- 2〜3列グリッドへ
- 料金行が縦並び（名前の下に価格）→ 横並び（名前 / 価格を左右）に変化
- デスクトップではグローバルナビが水平展開

### 6.3 モバイル限定・デスクトップ限定の表示切り替え
- 料金表などでは**モバイル専用UI**（縦・点線区切り）と**デスクトップ専用UI**（横並び）を用意し、画面幅で切り替える想定

## 8. 現状の強み・課題（リデザインの指針）

### 8.1 強み（維持すべき要素）
- 交互の背景色（白 / グレー）による視覚的リズム
- ページごとのアクセント色でサービス性格を伝える仕組み
- 豊富な施術例の画像（特にsinsページ）
- 段階的な情報提示（悩み → 特徴 → 注意 → 実例 → 料金 → CTA）
- 明確なCTA（電話 / LINE）

### 8.2 課題（リデザインで検討すべき要素）
- **フォント**: システムフォント依存のため、Webフォント導入で印象を統一できる余地あり
- **アイコン**: 記号文字とSVGが混在。アイコンセット統一で一貫性向上
- **画像プレースホルダー**: スタッフ・ギャラリーの一部に実写が入っていない
- **見出し色のばらつき**: ページにより青系・エメラルド系が混在 → 統一すべき
- **モバイルの料金表UI**: 縦並びと横並びで別マークアップになっており整理の余地あり
- **動きのバリエーション**: フェードインとホバー浮上のみ。パララックスや微細なマイクロインタラクションで質感向上の余地

### 8.3 技術的制約
- 画像は外部CDN（AVIF形式）経由。新規画像追加時は手動アップロード必要
- デザイントークン（色・サイズ・余白）は一元管理されており、トークン変更で全ページに反映可能
- 動的挙動（スムーススクロール / フェードイン / モバイルメニュー）は限定的な箇所のみ。他は静的レンダリング
