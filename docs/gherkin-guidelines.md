# Gherkin ガイドライン

## 概要

このドキュメントは、美容室successプロジェクトにおけるGherkinシナリオの記述規約をまとめたものです。
Gherkinは、ビジネス要件をテスト可能な仕様として記述するための言語です。

---

## 基本構文

```gherkin
Feature: 機能の名前

  Background:
    Given 共通の前提条件

  Scenario: シナリオの名前
    Given 前提条件
    When  操作・イベント
    Then  期待される結果
```

### キーワード一覧

| キーワード    | 意味                         |
|-------------|------------------------------|
| `Feature`   | テスト対象の機能               |
| `Background`| 各シナリオ共通の前提条件        |
| `Scenario`  | 1つのテストケース              |
| `Given`     | 前提条件（システムの初期状態）   |
| `When`      | ユーザーの操作やイベント        |
| `Then`      | 期待される結果・出力            |
| `And`       | 直前のステップと同じ種別の続き   |
| `But`       | 否定的な条件の追加              |
| `Examples`  | Scenario Outlineのデータテーブル|

## 記述ルール

### Feature

- 機能単位で1ファイルに1つ
- 名詞または動詞句で簡潔に記述する
- 必要であれば説明文を複数行で追加できる

```gherkin
# 良い例
Feature: モバイルメニューの開閉

# 悪い例
Feature: メニューがあって、ユーザーがそれを開いたり閉じたりできる機能のテスト
```

### Scenario

- 1つのシナリオは1つの振る舞いのみを検証する
- シナリオ名は「〜できる」「〜が表示される」など結果を表す形で書く
- 前提条件・操作・結果を明確に分離する

```gherkin
# 良い例
Scenario: ハンバーガーメニューをタップするとナビゲーションが表示される
  Given ユーザーがモバイル画面でトップページを開いている
  When  ハンバーガーメニューボタンをタップする
  Then  ナビゲーションメニューが表示される

# 悪い例（操作と結果が混在している）
Scenario: メニューのテスト
  Given ページを開く
  When  何かをクリックする
  Then  何かが起きる
```

### Given / When / Then

- **Given**: システムやユーザーの初期状態を記述する。過去形・現在完了形を使う
- **When**: ユーザーの1つのアクションを記述する。原則1ステップに1アクション
- **Then**: 観察可能な結果のみを記述する。実装の詳細には踏み込まない

```gherkin
# 良い例
Given ユーザーがトップページにアクセスしている
When  「予約する」ボタンをクリックする
Then  予約フォームが表示される

# 悪い例（実装詳細が含まれている）
Then  データベースにレコードが1件追加される
Then  HTTPステータス200が返される
```

### And / But

直前のキーワードの繰り返しを避けるために使う。意味は直前のキーワードに従う。

```gherkin
Given ユーザーがトップページを開いている
And   スクロール位置が先頭である
When  ページを下にスクロールする
And   ヘッダーエリアを通過する
Then  ヘッダーが固定表示になる
But   フッターは固定表示にならない
```

## このプロジェクトの記述例

### トップページの表示

```gherkin
Feature: トップページの表示

  Scenario: ページタイトルが正しく表示される
    Given ユーザーがトップページにアクセスしている
    Then  ページタイトルに「美容室success」が含まれる

  Scenario: ナビゲーションリンクが表示される
    Given ユーザーがトップページにアクセスしている
    Then  「メニュー」へのリンクが表示される
    And   「アクセス」へのリンクが表示される
    And   「お問い合わせ」へのリンクが表示される
```

### モバイルメニュー

```gherkin
Feature: モバイルメニューの操作

  Background:
    Given ユーザーがモバイル幅（375px）でページを開いている

  Scenario: メニューを開く
    Given モバイルメニューが閉じている
    When  ハンバーガーメニューボタンをタップする
    Then  ナビゲーションメニューが表示される
    And   ハンバーガーボタンのアイコンが閉じるアイコンに変わる

  Scenario: メニューを閉じる
    Given モバイルメニューが開いている
    When  閉じるボタンをタップする
    Then  ナビゲーションメニューが非表示になる

  Scenario: メニューのリンクをタップするとメニューが閉じる
    Given モバイルメニューが開いている
    When  「メニュー」リンクをタップする
    Then  ナビゲーションメニューが非表示になる
```

### CTAボタン

```gherkin
Feature: CTAボタンの表示

  Scenario Outline: 各ページにCTAボタンが表示される
    Given ユーザーが<ページ>にアクセスしている
    Then  「今すぐ予約する」ボタンが表示される

    Examples:
      | ページ         |
      | トップページ   |
      | メニューページ |
      | アクセスページ |
```

## アンチパターン

### UIの実装詳細を記述しない

```gherkin
# 悪い例（CSSクラス名が入っている）
Then  要素に "menu-open" クラスが付与される

# 良い例
Then  ナビゲーションメニューが表示される
```

### 1つのシナリオに複数の振る舞いを詰め込まない

```gherkin
# 悪い例
Scenario: ユーザーの操作
  When  メニューを開く
  And   リンクをクリックする
  And   ページが遷移する
  And   新しいページのタイトルを確認する
  And   戻るボタンを押す
  Then  元のページに戻る

# 良い例：シナリオを分割する
Scenario: メニューのリンクをクリックするとページが遷移する
  ...
Scenario: ブラウザの戻るボタンで前のページに戻れる
  ...
```

### Given に操作を含めない

```gherkin
# 悪い例
Given ユーザーがログインボタンをクリックした

# 良い例
Given ユーザーがログイン済みである
```
